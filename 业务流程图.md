## 四、端到端加密聊天流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          端到端加密聊天流程                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ==================== 密钥交换阶段 ====================                     │
│                                                                             │
│  用户 A                              用户 B                                  │
│    │                                   │                                    │
│    │  1. 生成密钥对                     │                                    │
│    │     (ECDH P-256)                  │                                    │
│    │                                   │                                    │
│    │  2. 上传公钥到服务器               │                                    │
│    │     POST /api/users/me/keys       │                                    │
│    │                                   │                                    │
│    │  3. 获取用户 B 的公钥              │                                    │
│    │     GET /api/users/{id}/publickey  │                                    │
│    │                                   │                                    │
│    │  4. ECDH 派生共享密钥             │                                    │
│    │     deriveSharedKey(privateKeyA, publicKeyB)                           │
│    │                                   │                                    │
│                                                                             │
│  ==================== 消息发送阶段 ====================                     │
│                                                                             │
│  用户 A                              服务器                                  │
│    │                                   │                                    │
│    │  1. 输入消息 "你好"               │                                    │
│    │                                   │                                    │
│    │  2. AES-GCM 加密                  │                                    │
│    │     encrypt("你好", sharedKey)    │                                    │
│    │     → "abc123..."                 │                                    │
│    │                                   │                                    │
│    │  3. 发送密文                       │                                    │
│    │     POST /api/messages/:pairId    │                                    │
│    │     { content: "abc123..." }      │                                    │
│    │──────────────────────────────────>│                                    │
│    │                                   │  4. 存储密文                        │
│    │                                   │     数据库: "abc123..."            │
│    │                                   │                                    │
│    │                                   │  5. WebSocket 转发                 │
│    │                                   │─────────────────────────────────>│ 用户 B
│    │                                   │                                    │
│                                                                             │
│  ==================== 消息接收阶段 ====================                     │
│                                                                             │
│  用户 B                                                      用户 A         │
│    │                                                           │           │
│    │  6. 收到 WebSocket 消息                                  │           │
│    │     { type: 'new_message', content: 'abc123...' }       │           │
│    │                                                         │           │
│    │  7. 获取用户 A 的公钥                                    │           │
│    │     GET /api/users/{id}/publickey                       │           │
│    │                                                         │           │
│    │  8. ECDH 派生共享密钥                                    │           │
│    │     deriveSharedKey(privateKeyB, publicKeyA)            │           │
│    │                                                         │           │
│    │  9. AES-GCM 解密                                        │           │
│    │     decrypt("abc123...", sharedKey)                     │           │
│    │     → "你好"                                            │           │
│    │                                                         │           │
│    │  10. 显示明文                                            │           │
│    │                                                         │           │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 五、时间胶囊流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              时间胶囊流程                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  创建阶段                                                                   │
│  ─────────                                                                 │
│  用户 A                                                                     │
│    │                                                                       │
│    │  1. 打开时间胶囊页面                                                   │
│    │                                                                       │
│    │  2. 点击"创建时间胶囊"                                                 │
│    │                                                                       │
│    │  3. 输入内容 + 选择触发时间                                             │
│    │                                                                       │
│    │  4. API: POST /api/capsules                                           │
│    │     {                                                                 │
│    │       pairId: "xxx",                                                  │
│    │       receiverId: "user_b_id",                                        │
│    │       content: "加密内容",                                             │
│    │       triggerTime: "2024-02-14T00:00:00Z"                            │
│    │     }                                                                 │
│    │                                                                       │
│    │  5. 服务器存储胶囊，状态 = pending                                     │
│    │                                                                       │
│  等待阶段                                                                   │
│  ─────────                                                                 │
│  定时任务 (每分钟)                                                          │
│    │                                                                       │
│    │  1. 查询 status='pending' 且 triggerTime <= now 的胶囊                 │
│    │                                                                       │
│    │  2. 更新状态为 delivered                                                │
│    │                                                                       │
│    │  3. 发送通知                                                           │
│    │     - FCM (Android) 或 Server酱 (iOS)                                 │
│    │     - 通用文案："你有一条来自过去的回声"                                 │
│    │                                                                       │
│  接收阶段                                                                   │
│  ─────────                                                                 │
│  用户 B                                                                     │
│    │                                                                       │
│    │  4. 收到通知                                                           │
│    │                                                                       │
│    │  5. 打开 App，进入时间胶囊页面                                          │
│    │                                                                       │
│    │  6. 点击"打开时间胶囊"                                                  │
│    │                                                                       │
│    │  7. API: POST /api/capsules/:id/open                                  │
│    │                                                                       │
│    │  8. 服务器更新状态为 read                                               │
│    │                                                                       │
│    │  9. 显示胶囊内容（带开箱动画）                                          │
│    │                                                                       │
└─────────────────────────────────────────────────────────────────────────────┘
```

```
│  if (消息数 === 1000) → 生成"千言万语"里程碑                                 │
```

## 十二、配对功能实现详解

### 12.1 数据库模型

```sql
Pair 表:
├── id (String, 主键, cuid)
├── status (String, 默认 "active")
│   ├── active: 正常配对
│   └── inactive: 已解除
├── userAId (String, 用户A的ID, 外键)
├── userBId (String, 用户B的ID, 外键)
├── createdAt (DateTime, 创建时间)
│
└── 约束
    ├── @@unique([userAId, userBId])  -- 唯一约束，避免重复配对
    ├── userAId < userBId             -- 规范化，确保唯一性
    └── 外键关联 User 表
```

### 12.2 后端 API

#### 创建配对
```
POST /api/pairs
Authorization: Bearer <token>
Content-Type: application/json

{
  "partnerId": "对方用户ID"
}

响应:
{
  "pair": {
    "id": "cmqxxx",
    "status": "active",
    "userAId": "user_a_id",
    "userBId": "user_b_id",
    "userA": { ... },
    "userB": { ... },
    "createdAt": "2024-01-01T00:00:00Z"
  }
}
```

处理流程:
```
1. 验证用户已登录 (JWT)
   ↓
2. 检查 partnerId 是否有效
   ↓
3. 检查是否已与对方配对
   - 查询 Pair 表中 userAId 和 userBId 的组合
   - 如果已存在，返回现有配对
   ↓
4. 规范化: 确保 userAId < userBId
   - 如果 userId < partnerId: userAId = userId
   - 如果 userId > partnerId: userAId = partnerId
   ↓
5. 创建 Pair 记录
   ↓
6. 返回配对信息
```

#### 获取配对列表
```
GET /api/pairs
Authorization: Bearer <token>

响应:
{
  "pairs": [
    {
      "id": "cmqxxx",
      "status": "active",
      "userA": { "id": "...", "username": "test", ... },
      "userB": { "id": "...", "username": "alice", ... },
      "createdAt": "2024-01-01T00:00:00Z"
    }
  ]
}
```

处理流程:
```
1. 验证用户已登录
   ↓
2. 查询所有配对
   - WHERE userAId = userId OR userBId = userId
   - AND status = 'active'
   ↓
3. 包含对方用户信息
   ↓
4. 返回配对列表
```

### 12.3 前端流程

```
用户 A 操作:
┌─────────────────────────────────────────────────────────────┐

│     └── 搜索用户 B (GET /api/users/search?q=xxx)            │
```

### 12.4 配对状态说明

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| active | 正常配对 | 聊天、共享日记、共享相册 |
| inactive | 已解除配对 | 无 |
